---

## Rules of the Game

1. **Sustainability > intensity.** Missing a day is fine; missing a week needs recovery plan.
2. **Retrieval > re-reading.** Active recall beats passive review every time.
3. **Explain before code.** Verbal pseudocode + Big O before writing any Swift.
4. **Timed solo solving.** Track how long real problem-solving takes — this is the interview gap.
5. **Weekly review is sacred.** Sunday 10 min, non-negotiable. Without it, retention collapses.
